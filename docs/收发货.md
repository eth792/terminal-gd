# 收发货 RPA 工具 - UI交互逻辑文档

## 1. 整体交互流程

```
登录页面 ──→ 主页面（功能选择） ──→ 收发货运行页面 ──→ 执行完成/返回主页
   ↓                ↓                    ↓
 记住密码        功能卡片              实时日志
 错误提示        用户信息              进度反馈
                退出登录              错误处理
```

---

## 2. 登录模块交互逻辑

### 2.1 页面初始化
**触发时机**: 应用启动  
**交互流程**:
1. 检查本地是否有保存的登录凭据
   - **有** → 自动填充用户名，密码框保持空白
   - **无** → 显示空白登录表单
2. 显示登录界面（Material Design风格）
3. 光标自动定位到用户名输入框

### 2.2 用户输入交互
**输入框行为**:

| 操作 | 交互反馈 |
|------|---------|
| 聚焦输入框 | 输入框边框高亮（主题色） |
| 输入内容 | 实时显示输入内容（密码框显示圆点） |
| 输入错误格式 | 失焦时显示格式错误提示（红色文字） |
| 清空内容 | 显示清除按钮（×） |

**键盘交互**:
- `Tab` 键: 在用户名 → 密码 → 记住密码 → 登录按钮间切换
- `Enter` 键: 在任何输入框中按下即触发登录

### 2.3 登录按钮状态

| 状态 | 视觉呈现 | 交互行为 |
|------|---------|---------|
| **默认状态** | 蓝色按钮，白色文字 "登录" | 可点击 |
| **禁用状态** | 灰色按钮，浅灰文字 | 不可点击（用户名或密码为空时） |
| **加载中** | 蓝色按钮 + 加载动画（转圈图标） | 不可点击，显示"登录中..." |
| **悬停状态** | 深蓝色背景 | 鼠标指针变为手型 |

### 2.4 登录流程交互

```
点击登录按钮
    ↓
表单验证
    ├── 验证失败 → 显示错误提示（输入框下方红色文字）
    └── 验证通过 → 按钮进入加载状态
            ↓
        发送登录请求
            ├── 成功 (200ms后)
            │   ├── 显示成功提示（绿色勾图标 + "登录成功"）
            │   ├── 保存登录状态（如果勾选"记住密码"）
            │   └── 渐隐过渡到主页面（300ms动画）
            │
            └── 失败
                ├── 账号密码错误 → 显示错误提示（"用户名或密码错误"）
                ├── 网络错误 → 显示错误提示（"网络连接失败，请重试"）
                └── 其他错误 → 显示通用错误提示
                ↓
                密码框清空，光标定位到密码框
                按钮恢复默认状态
```

### 2.5 记住密码交互
- **默认状态**: 复选框未选中
- **点击行为**: 切换选中/未选中状态
- **保存逻辑**: 
  - 选中 + 登录成功 → 本地加密保存用户名
  - 未选中 → 清除本地保存的凭据

## 3. 主页面模块交互逻辑

### 3.1 页面布局

```
┌────────────────────────────────────────────────────────┐
│  导航栏（顶部）                                         │
│  ┌──────────────────────────────────────────────────┐  │
│  │  RPA 自动化工具    [用户信息] [退出登录]          │  │
│  └──────────────────────────────────────────────────┘  │
├────────────────────────────────────────────────────────┤
│  主内容区                                               │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐      │
│  │  收发货    │  │  自动填报  │  │ 表格提取   │      │
│  │  管理      │  │            │  │            │      │
│  │  [图标]    │  │  [图标]    │  │  [图标]    │      │
│  │  点击运行  │  │  点击运行  │  │  点击运行  │      │
│  └────────────┘  └────────────┘  └────────────┘      │
└────────────────────────────────────────────────────────┘
```

### 3.2 页面进入动画
**触发时机**: 从登录页面跳转过来  
**动画效果**:
1. 登录页面淡出（200ms）
2. 主页面从底部滑入（300ms，缓动函数: ease-out）
3. 功能卡片依次从左到右淡入（每个间隔100ms）

### 3.3 功能卡片交互

#### 卡片状态

| 状态 | 视觉效果 | 交互行为 |
|------|---------|---------|
| **默认** | 白色背景，灰色边框，功能图标+标题+描述 | - |
| **悬停** | 轻微上浮（4px），阴影加深，边框变为主题色 | 鼠标变手型 |
| **按下** | 轻微下沉（2px），阴影减弱 | - |
| **点击** | 涟漪动画（从点击位置扩散），300ms后跳转 | 跳转到运行页面 |
| **禁用** | 灰色背景，半透明，显示"即将上线" | 不可点击，鼠标变禁止图标 |

#### 卡片内容结构
```
┌─────────────────────┐
│     [功能图标]       │  ← 64x64px，主题色
│                     │
│    收发货管理        │  ← 标题，18px，加粗
│                     │
│  自动识别单据并填报  │  ← 描述，14px，灰色
│                     │
│     点击运行 →      │  ← 行动召唤，12px，主题色
└─────────────────────┘
```

### 3.4 用户信息交互
**位置**: 右上角  
**默认显示**: 用户名 + 头像图标  
**悬停效果**: 
- 显示下拉菜单（150ms淡入）
- 菜单项:
  - 个人信息（暂未开放，灰色）
  - 设置（暂未开放，灰色）
  - ──────（分割线）
  - **退出登录**（红色文字，可点击）

### 3.5 退出登录交互流程
```
点击"退出登录"
    ↓
弹出确认对话框（模态）
    ┌─────────────────────────┐
    │  确认退出               │
    │  您确定要退出登录吗？    │
    │  [ 取消 ]  [ 确定 ]     │
    └─────────────────────────┘
    ↓
    ├── 点击取消 → 关闭对话框，留在当前页面
    └── 点击确定 → 
            ↓
        清除登录状态
        显示"正在退出..."（300ms）
        渐隐过渡到登录页面（300ms）
```

### 4.2 页面进入动画
1. 从主页面点击卡片后
2. 主页面从左侧滑出（300ms）
3. 运行页面从右侧滑入（300ms）
4. 配置区域和日志区域依次淡入（各200ms）
5. 日志输出第一条消息: `[14:23:45] INFO 系统就绪，等待开始...`

### 4.3 返回按钮交互
**位置**: 左上角  
**样式**: `← 返回主页`（文字按钮）  
**交互流程**:
```
点击返回按钮
    ↓
检查运行状态
    ├── 未运行/已完成 → 直接返回主页（300ms过渡动画）
    └── 运行中 → 弹出确认对话框
            ┌─────────────────────────────┐
            │  确认离开                    │
            │  任务正在执行中，确认返回吗？ │
            │  (任务将在后台继续执行)       │
            │  [ 取消 ]  [ 确定返回 ]      │
            └─────────────────────────────┘
            ↓
            ├── 点击取消 → 关闭对话框
            └── 点击确定 → 返回主页，任务继续
```

### 4.4 运行按钮交互

#### 按钮状态流转

```
待运行 ──点击──→ 运行中 ──完成/错误──→ 已完成 ──重置──→ 待运行
  ↓                 ↓                     ↓
[开始运行]      [执行中...]         [重新运行] / [查看结果]
蓝色背景         绿色背景             蓝色背景 / 灰色背景
可点击          不可点击              可点击
```

#### 详细状态定义

| 状态 | 文字 | 背景色 | 图标 | 点击行为 |
|------|------|--------|------|---------|
| **待运行** | "开始运行" | 蓝色 | ▶ | 开始执行流程 |
| **运行中** | "执行中..." | 绿色 + 脉冲动画 | ⏸（加载动画） | 不可点击 |
| **已完成（成功）** | "运行完成" | 绿色 | ✓ | 显示结果摘要对话框 |
| **已完成（部分成功）** | "部分完成" | 黄色 | ⚠ | 显示详细错误报告 |
| **已完成（失败）** | "运行失败" | 红色 | ✗ | 显示错误信息 |
| **暂停中** | "已暂停" | 灰色 | ▶ | 继续执行（如果支持） |

### 4.5 步骤区域交互逻辑

#### 4.5.1 步骤1: 扫描纸张获取JSON数据

**交互元素**:
1. **上传图片按钮**
   - 点击 → 打开文件选择器
   - 支持格式: `.jpg`, `.jpeg`, `.png`, `.pdf`
   - 支持多选
   - 拖拽交互:
     ```
     拖拽文件进入区域
         ↓
     区域边框虚线高亮（蓝色）
         ↓
     松开鼠标
         ├── 文件格式正确 → 添加到列表，显示文件名和大小
         └── 文件格式错误 → 显示错误提示（红色）
     ```

2. **扫描仪扫描按钮**
   - 点击 → 检测扫描仪设备
   - 未检测到 → 显示提示"未检测到扫描仪设备"
   - 检测到 → 打开扫描仪对话框
   - 扫描完成 → 添加到文件列表

3. **文件列表显示**
   ```
   ┌────────────────────────────────────┐
   │ ✓ 验收单_001.jpg     (2.3MB)  [×] │  ← 可删除
   │ ✓ 验收单_002.pdf     (1.8MB)  [×] │
   │ ✓ 到货验收单.png     (3.1MB)  [×] │
   │                                    │
   │ 共 3 个文件，总大小 7.2MB          │
   └────────────────────────────────────┘
   ```
   - 每个文件项:
     - 悬停 → 背景色变浅蓝
     - 点击 [×] → 移除文件（带淡出动画）
     - 点击文件名 → 预览图片/PDF

4. **状态显示**
   - 待执行: `[待执行]` 灰色标签
   - 执行中: `[执行中...]` 绿色标签 + 进度条
     ```
     OCR 识别中... ████████░░░░ 67%
     已处理 2/3 个文件
     ```
   - 完成: `[已完成]` 蓝色标签 + 结果概览
     ```
     ✓ 成功识别 3 个文件
     识别数据质量评分: 92/100
     ```
   - 错误: `[错误]` 红色标签 + 错误信息

**自动执行逻辑**:

**用户交互**:
- 点击`[查看详情]` → 打开数据预览对话框
  - 显示处理后的结构化数据（表格形式）
  - 高亮显示异常数据（黄色背景）
  - 支持手动编辑异常数据

### 4.6 日志系统交互

#### 4.6.1 日志过滤器

**过滤按钮组**（Tab标签样式）:
```
[全部] [INFO] [WARNING] [ERROR] [SUCCESS]
  ↓      ↓       ↓         ↓        ↓
 默认   蓝色    黄色      红色     绿色
```

**交互行为**:
- 点击某个级别 → 只显示该级别的日志
- 点击`[全部]` → 显示所有日志
- 当前选中的标签: 实心背景 + 白色文字
- 未选中的标签: 空心边框 + 主题色文字
- 每个标签显示数量徽章:
  ```
  [ERROR ②]  [WARNING ⑤]
  ```

#### 4.6.2 日志工具栏

| 工具 | 交互行为 |
|------|---------|
| **[清空日志]** | 点击 → 确认对话框 → 清空所有日志 |
| **[导出日志]** | 点击 → 文件保存对话框 → 导出为.txt或.json |
| **[搜索]** | 点击 → 展开搜索框 → 高亮匹配的日志行 |
| **[自动滚动 ✓]** | 切换开关 → 新日志时自动滚动到底部 |

#### 4.6.3 日志条目显示

**格式**:
```
[时间] [级别] 消息内容
[14:23:45] INFO  系统就绪，等待开始...
[14:24:10] SUCCESS 成功加载 3 个文件
[14:24:15] WARNING 文件 "验收单_002.pdf" 质量较低，可能影响识别准确率
[14:24:20] ERROR 文件 "到货单.jpg" OCR识别失败: 图片模糊
```

**视觉样式**:
- INFO: 灰色文字，白色背景
- SUCCESS: 绿色文字，浅绿背景
- WARNING: 黄色文字，浅黄背景
- ERROR: 红色文字，浅红背景

**交互行为**:
- 悬停日志行 → 显示完整时间戳（带毫秒）
- 点击日志行 → 展开详细信息（如果有）
  ```
  [14:24:20] ERROR 文件 "到货单.jpg" OCR识别失败: 图片模糊
      ↓ 点击展开
  [14:24:20] ERROR 文件 "到货单.jpg" OCR识别失败: 图片模糊
      详细信息:
      - 文件路径: /Users/xxx/到货单.jpg
      - 错误代码: OCR_001
      - 建议: 请提高图片分辨率或重新扫描
      [复制错误信息] [查看帮助文档]
  ```

#### 4.6.4 实时日志推送

**推送策略**:
- 使用虚拟滚动，最多显示1000条日志
- 超过1000条自动清除最早的日志
- 新日志从底部插入，带淡入动画（100ms）
- 如果用户正在查看历史日志（滚动条不在底部）:
  - 不自动滚动
  - 显示提示: `↓ 有新日志 (5条)` （点击跳到底部）

## 5. 全局交互规范

### 5.1 加载状态

**场景**: 任何需要等待的操作（登录、数据加载、文件上传等）

**视觉设计**:
```
┌─────────────────┐
│                 │
│   ⏳ 加载中...   │  ← 旋转动画
│                 │
└─────────────────┘
```

**时长控制**:
- < 1秒: 不显示加载状态
- 1-3秒: 显示简单加载动画
- > 3秒: 显示加载动画 + 进度百分比

### 5.2 错误提示

**类型与显示方式**:

| 错误类型 | 显示位置 | 持续时间 | 交互 |
|---------|---------|---------|------|
| **表单验证错误** | 输入框下方 | 用户修改前一直显示 | 点击输入框清除 |
| **操作失败提示** | 页面顶部Toast | 3秒后自动消失 | 可手动关闭 |
| **系统级错误** | 模态对话框 | 用户手动关闭 | 必须点击确认 |
| **网络错误** | 页面顶部横幅 | 网络恢复后消失 | 可手动重试 |

**Toast提示设计**:
```
┌────────────────────────────────────┐
│ ✗ 操作失败: 网络连接超时  [重试] [×]│  ← 红色背景
└────────────────────────────────────┘
       从顶部滑入（200ms）
       3秒后自动滑出（200ms）
```

### 5.3 成功反馈

**Toast设计**:
```
┌────────────────────────────────────┐
│ ✓ 操作成功: 数据已保存          [×]│  ← 绿色背景
└────────────────────────────────────┘
```

**时长**: 2秒后自动消失

### 5.4 确认对话框

**统一样式**:
```
┌─────────────────────────────────┐
│  [⚠] 标题                        │
│  ─────────────────────────────  │
│  描述文字说明                    │
│                                 │
│              [ 取消 ] [ 确定 ] │
└─────────────────────────────────┘
```

**交互规则**:
- 按 `Esc` 键 = 点击"取消"
- 按 `Enter` 键 = 点击"确定"
- 点击背景遮罩 = 点击"取消"

### 5.5 响应式布局

**断点设计**:
- 宽度 ≥ 1200px: 双栏布局（配置40% + 日志60%）
- 宽度 800-1200px: 双栏布局（配置50% + 日志50%）
- 宽度 < 800px: 单栏布局，配置和日志上下排列（Tab切换）

### 5.6 快捷键支持

| 快捷键 | 功能 | 生效页面 |
|-------|------|---------|
| `Ctrl/Cmd + R` | 开始运行/重新运行 | 运行页面 |
| `Ctrl/Cmd + L` | 清空日志 | 运行页面 |
| `Ctrl/Cmd + F` | 搜索日志 | 运行页面 |
| `Ctrl/Cmd + ←` | 返回上一页 | 所有页面 |
| `Esc` | 关闭对话框/取消操作 | 所有页面 |

### 5.7 无障碍支持 (A11y)

- 所有交互元素支持键盘Tab导航
- 按钮和链接有清晰的焦点状态（蓝色边框）
- 图片和图标提供alt文字描述
- 颜色编码信息同时提供文字标签（如ERROR文字+红色）
- 支持屏幕阅读器（ARIA标签）

## 6. 性能优化交互

### 6.1 大文件上传

**文件大小限制**:
- 单文件: 最大20MB
- 总大小: 最大100MB

**超出限制交互**:
```
选择文件 > 20MB
    ↓
显示错误提示
    ┌──────────────────────────────┐
    │  ✗ 文件过大                   │
    │  "验收单.pdf" (25MB) 超出     │
    │  20MB限制，请压缩后上传       │
    │  [ 了解如何压缩 ] [ 确定 ]    │
    └──────────────────────────────┘
```

**大文件上传进度**:
```
┌────────────────────────────────┐
│ 上传中: 验收单.pdf             │
│ ████████████░░░░ 75% (15MB/20MB)│
│ 速度: 2.3MB/s  剩余: 2秒       │
│           [ 取消上传 ]          │
└────────────────────────────────┘
```

### 6.2 批量数据处理

**分批处理反馈**:
- 每处理10条记录更新一次进度
- 每秒最多更新UI 2次（避免频繁渲染）
- 使用虚拟列表展示大量数据（>100条）

### 6.3 防抖和节流

**应用场景**:
- 搜索输入: 防抖300ms
- 窗口大小调整: 节流200ms
- 滚动事件: 节流100ms

## 7. 边界情况处理

### 7.1 网络断开

**检测**: 每5秒检测一次网络状态

**交互**:
```
网络断开
    ↓
顶部显示警告横幅

    ↓
```

```
    │  [ 继续使用 ]                │
```

```
│      [ 上传文件 ]              │
│                                │
└────────────────────────────────┘
```

### 7.4 系统资源不足

**检测**: 监控内存使用率

**交互**:

```
内存使用 > 85%

显示警告Toast
```
